O DataStore implementa **209 métodos de DataFrame do pandas** para oferecer compatibilidade total com a API. Seu código existente em pandas funciona com mudanças mínimas.

<div id="approach">
  ## Abordagem de compatibilidade
</div>

```python
# Migração típica - apenas mude o import
- import pandas as pd
+ from chdb import datastore as pd

# Seu código funciona sem alterações
df = pd.read_csv("data.csv")
result = df[df['age'] > 25].groupby('city')['salary'].mean()
```

**Princípios-chave:**

* Todos os 209 métodos do DataFrame do pandas implementados
* Avaliação preguiçosa para otimização de SQL
* Encapsulamento automático de tipos (DataFrame → DataStore, Series → ColumnExpr)
* Operações imutáveis (sem `inplace=True`)

**Exemplos:**

```python

print(ds.columns)    # Index(['name', 'age', 'city', 'salary', 'dept'])
print(ds.dtypes)     # name: object, age: int64, ...
print(ds.empty)      # False
```

<div id="differences">
  ## Principais diferenças em relação ao Pandas
</div>

<div id="return-types">
  ### 1. Tipos de retorno
</div>

```python
# O Pandas retorna Series
pdf['col']  # → pd.Series

# DataStore retorna ColumnExpr (lazy)
ds['col']   # → ColumnExpr
```

### 2. Execução preguiçosa {#lazy-execution}

```python
# Operações do DataStore são preguiçosas (execução adiada)
result = ds.filter(ds['age'] > 25)  # Ainda não executado
df = result.to_df()  # Executado aqui
```

<div id="no-inplace-parameter">
  ### 3. Sem parâmetro inplace
</div>

```python
# Pandas
df.drop(columns=['col'], inplace=True)

# DataStore (sempre retorna um novo objeto)
ds = ds.drop(columns=['col'])
```

<div id="comparing-results">
  ### 4. Comparação de resultados
</div>

```python
# Use to_pandas() para comparação
pd.testing.assert_frame_equal(
    ds.to_pandas(),
    expected_df
)
```
